Java 线程与锁机制

1. Java 如何开启线程

线程与进程的区别：进程是操作系统进行资源分配的最小单元；进程是操作系统进行任务分配的最小单元，线程隶属于进程。

开启线程的四种方式：

  • 继承Thread类，重写run方法；
  • 实现Runnable接口，实现run方法；
  • 实现Callable接口，实现Call方法；
  • 通过线程池开启线程。

线程安全是指什么：多线程情况下，调用同一个对象都能得到正确的返回结果，即线程安全。
怎样保证线程安全：加锁：1. JVM提供的锁，即Syncchronized关键字； 2. JDK提供的各种锁Lock

3. Java线程锁机制是怎样的？偏向锁、轻量级锁、重量级锁有什么区别？锁机制是如何升级的？

  • Java中的锁就是在对象的MarkWord中记录一个锁状态。无锁，偏向锁，轻量级锁，重量级锁对应不同的锁状态。
  • Java的锁机制就是根据资源竞争的激烈程度不断进行锁升级的过程。
  • 根据资源竞争的激烈程度，来进行锁的升级。
- 偏向锁：相当于一个对象中留了个门牌，表示请大家（即线程）有序访问，想要后续访问对象中的资源需要排队，像是在门卫中登记下，登记成功后才可访问资源；但这时候还没有上锁，只是告诉大家，我已经登记过了，正在访问资源。
- 轻量级锁：随着后面排队的人越来越多，看到已经有人在门卫处登记了，只有等上一位出来，才可有下一位登记进去访问资源，所以大家会不断自旋（即不断尝试获取登记），等上一位出来后，便进去登记占有门牌标识，然后访问资源，所以轻量级锁也叫做“自旋锁”。
- 重量级锁： 当有很多的人排队时，大家都在自旋尝试获有门牌标识，那么便是对cpu的一种浪费，当竞争过于激烈的时候，Jvm便会上报给操作系统，由操作系统霸道进行排队，等操作系统通知（每一步是操作系统来执行，cpu效率会降低）。轻量级锁，不断自旋，提高获取锁的效率。

4. 谈谈你对AQS的理解？AQS如何实现可重入锁？

  • AQS是一个Java线程的同步框架，是JDK中很多锁工具的核心实现（阅读源码）
  • 在AQS中，维护了一个信号量state和一个线程组成的双向链表队列。其中，这个线程队列，就是用来给线程排队的，而state就像是一个红绿灯，用于控制线程排队或放行的，在不同的场景下，有不同的意义。
  • 在可重入锁这个场景下，state就是表示加锁的次数，0表示无锁，每加一次锁，state就加1，有释放锁就减1。
